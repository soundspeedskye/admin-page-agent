# I18n Rule

다국어는 i18next와 react-i18next를 사용한다.

## Files

- `app/i18n/index.ts`
- `app/i18n/{language}/translation.json`
- `app/i18n/i18next.d.ts`

## Rules

- 기본 언어와 fallback은 split input의 `defaultLanguage`를 따른다.
- 생성 언어는 split input의 `supportedLanguages`를 따른다.
- UI 문구는 `useTranslation()`의 `t()`로 렌더링한다.
- schema 에러도 가능한 경우 `t()`를 받는 factory로 처리한다.
- 전역 language store 변경 시 `i18n.changeLanguage()`를 호출한다.
- key는 domain 기준으로 묶는다.

하드코딩 문구는 임시 테스트가 아니면 남기지 않는다.
